把你想用作“真实商品图”的图片放到这里（jpg/png/webp 都行）。

最推荐：按目录投放（更可控、更像电商）

demo_images/
  products/              # 你自己准备的“商品图”（最重要：会用来生成商品 + 封面图）
    digital/             # 数码（手机/耳机/键盘/相机等）
    home/                # 家居（收纳/小家电/灯具等）
    book/                # 书籍（书封/套装/文具等）
  cover/                 # 商品封面图素材（优先使用）
    digital/             # 数码（手机/耳机/键盘/相机等）
    home/                # 家居（收纳/小家电/灯具等）
    book/                # 书籍（书封/套装/文具等）
  banner/                # 首页轮播图素材（优先使用）
  ad/                    # 首页广告位素材（优先使用）
  (也可以把图片直接放在 demo_images/ 根目录，脚本会作为兜底素材使用)

关于 demo_images/products：
- 只要 products/ 下存在图片，seed_demo 会优先用这些图来“创建商品 + 生成封面图”，并根据文件名/同名 .json 生成更贴图的标题/副标题/规格简介。
- 说明：纯靠图片内容自动识别“这是什么商品”（图像识别）需要额外引入视觉模型/服务，项目目前没做；这里采用更稳的方式：你用“目录 + 文件名 + 可选元数据”告诉脚本这张图是什么。

文件命名建议（不写 json 也能跑，但写了会更像真电商）：
- 推荐用双下划线分段：品牌__型号__版本__颜色__其他.jpg
  示例：
    Apple__iPhone 15 Pro__256G__原色钛金.jpg
    小米__空气炸锅__4L__白色.jpg
    深入理解计算机系统__第3版__中文版.jpg
- 可选：在文件名里写价格（脚本会尝试解析）：__¥199 或 __1999

同名 .json 元数据（强烈推荐，能精确控制简介）：
- 和图片同名：Apple__iPhone 15 Pro__256G.jpg + Apple__iPhone 15 Pro__256G.json
- JSON 示例：
  {
    "classification": "数码",
    "title": "Apple iPhone 15 Pro 256G 原色钛金",
    "subtitle": "A17 Pro｜120Hz｜4800万像素｜Type-C",
    "brand": "Apple",
    "model": "iPhone 15 Pro",
    "variant": "256G 原色钛金",
    "price": 7999,
    "repertory": 30,
    "specs": ["屏幕：6.1英寸", "存储：256G", "接口：USB-C"],
    "highlights": ["全新正品（演示）", "现货速发（演示）", "支持7天无理由（演示）"],
    "tag_titles": ["热门", "包邮"]
  }

脚本会自动：
- 将素材裁剪/适配为封面（1200x800）、轮播（1600x520）、广告（1200x480）
- 做“白底电商风格”处理（背景提亮 + 前景居中 + 阴影）
- 叠加商品标题/活动文案，保证缩略图可读

素材建议（你自己准备图时，效果会好很多）：
- 选“商品主图/白底/棚拍/packshot/studio”这类图，尽量避免风景/人物/复杂背景
- 分辨率建议 ≥ 1200px（越清晰越好），主体居中，四周留白会更适合裁剪
- 避免带明显品牌 Logo/水印/平台标识（演示更干净）

用法：在 code\server 目录执行
  pixi run seed_demo
然后刷新前端首页即可。

如果你暂时不想用 products/ 里的图片生成商品（只想用脚本内置商品模板），可以：
  pixi run seed_demo -- --ignore-product-images

可选：如果你还想用脚本自动下载图片（兜底用，不影响你自己放的图）
  pixi run seed_demo_remote
或强制重下：
  pixi run seed_demo_remote_force
